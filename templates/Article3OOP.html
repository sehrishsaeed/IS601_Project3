Object Oriented Programming

4 important concepts to know


Polymorphism, Encapsulation, Abstraction, & Inheritance

What do all these mean?

There are 4 basic principles of Object Oriented Programming

Polymorphism

  • Polymorphism gives a way to use a class exactly like its parent so there's no confusion with mixing types. It is also is used so that a variable, function, or object can take multiple forms. Polymorphism uses a single symbol to represent multiple different types. This typically happens by defining a parent interface to be reused. Child classes implement their own version of these methods.

    Encapsulation

  • Encapsulation is achieved when each object keeps its state private, inside a class. Other objects don't have access to this state. Instead, they can only call a list of public functions which are called methods.

    Abstraction

  • Abstraction is basically the next step of encapsulation. In using OOP, Abstraction is your solution to programs that are very large and involve a lot of different objects that communicate with each other.

    "Using abstract class/Interface we express the intent of the class rather than the actual implementation.     Inheritance

  • One way to think about inheritance is that it aims to express “is-a” and/or “has-a” relationship between two objects. The benefit is that it provides re-usability of a code.
