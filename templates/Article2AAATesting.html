Arrange-Act-Assert or AAA for short

The importance and why we should use it

More on AAA

Arrange: where you enter the variables being tested

Arrange steps should set up the test case.

Act: checks the behavior

Act steps should cover the main part to be tested, ie your calculations in the calc example

Assert: The outcome

Assertions will ultimately determine if the test passes and give you an answer on whether or not your tests work properly. This is the final stage of the test.

Applying these tests in a Calculator

What exactly does it mean to test something?

The point of a test is to indicate whether something has passed or failed. Essentially, a test determines if a certain behavior functions properly. There are many different types of tests in tech but we'll focus on AAA testing here. There's a lot of testing that happens in programming and as you learn in most web development classes, testing ensures that your program is of top quality and is meeting all set requirements. Below is an example of AAA testing that we used in webstorm to create calculator tests. In the example you'll see multiplication being utilized.

AAA stands for Arrange-Act-Assert

Arrange-Act-Insert, better known as AAA is important because in a program it will ensure that tests are working properly. They will look at individual activity within a code and make sure that it is working correctly. It is a pattern for writing good tests. For example, according to automationpanda.com, (which is a great website to learn more about these topics) "The Arrange step creates a variable named “negative” for testing. The Act step calls the “abs” function using the “negative” variable and stores the returned value in a variable named “answer.” The Assert step verifies that “answer” is a positive value."

  • In the calculator example Arrange creates the variables "1.0" and "2.0" as "mynumbers" which indicates that they are the variables being multiplied together.
  • In the next step, which is Act, the "Multiply" function using the "mynumbers" variable and stores the new value after the equation is solved.
  • The final step in the test which is the Assert, verified that "multiplication.get_result" will give the result of "2.0".

    • The Different Steps

    Arrange

    This is the first step in AAA Testing. It is important to answer the questions below before we begin.

  • Is there a need for special objects or settings?
  • Will it require a web application to use?
  • Will it involve using a database?
  • If there is a database how will it be connected?

    • Act

    Once we complete the arrange portion of the test, the next step is the Act part. In this part you can create objects/methods relating to your function. In the example of the calculator, we made the object name as "calc". In this case we are using addition as the method here.

    Assert

    This is the last portion of the AAA testing. Here you should get an outcome that you are expecting. For example, this is where you would see if the test has passed or failed your program. In the example of the calculator, we added two number expecting a certain result and if the answer did not match up to what was expected, this test would fail. If the calculation was correct then in this phase we would see it pass.
